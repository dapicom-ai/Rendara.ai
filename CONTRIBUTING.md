# Contributing to Rendara

Thank you for your interest in contributing. Rendara is an open-development project — the source is public, and community contributions are welcome.

## License Agreement

Rendara is licensed under the [Business Source License 1.1](LICENSE). By submitting a pull request or any other contribution, you agree that your contribution will be licensed under the same terms. You confirm that you have the right to grant this license and that your contribution is your own original work.

## What We Welcome

- Bug fixes
- Documentation improvements
- New MCP server integrations or examples
- Visualisation improvements (new chart types, rendering fixes)
- UI/UX improvements
- Performance improvements
- Tests

## What to Discuss First

For larger changes — new features, architectural changes, or anything that affects the data model — please open a GitHub Issue before writing code. This avoids wasted effort if the direction doesn't fit the project.

## How to Contribute

1. Fork the repository
2. Create a branch from `main` with a descriptive name (e.g. `fix/chart-overflow`, `feat/line-chart-labels`)
3. Make your changes
4. Test locally — run the backend and frontend and verify your change works end to end
5. Open a pull request against `main` with a clear description of what you changed and why

## Code Style

- **Frontend**: TypeScript, Tailwind CSS, functional React components
- **Backend**: Python, type hints where practical, async FastAPI patterns
- Keep changes focused — one concern per PR
- Do not introduce new dependencies without discussing in the issue first

## Reporting Bugs

Open a GitHub Issue with:
- What you expected to happen
- What actually happened
- Steps to reproduce
- Browser/OS/Node/Python versions if relevant

## Questions

Open a GitHub Discussion or Issue. We are happy to help.
